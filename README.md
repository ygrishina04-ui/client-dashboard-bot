# MVP дашборда клиентского портфеля

## Как работает
1. В Telegram-бот загружаются 3 Excel-файла:
   - Заказы
   - Запросы
   - Клиентский портфель
2. Бот читает файлы, считает показатели и формирует `output/dashboard.html`.
3. В ответ бот отправляет готовый HTML-файл.

## Переменные окружения
- `BOT_TOKEN` — токен Telegram-бота от BotFather.
- `DASHBOARD_URL` — публичная ссылка Render-сервиса, например `https://your-service.onrender.com/dashboard`.
- `PORT` — порт Render, обычно Render задает автоматически.

## Запуск локально
```bash
pip install -r requirements.txt
export BOT_TOKEN=ваш_токен
export DASHBOARD_URL=http://localhost:10000/dashboard
python bot.py
```

Локально дашборд откроется по адресу `http://localhost:10000/dashboard`. На Render будет ссылка вашего сервиса.

## Правила расчета

### Заказы в работе
Статусы:
- В работе
- До границы
- ЖД
- Жд прямое
- Море
- Ожидание выхода по ЖД
- Оплата за товар
- Размещение
- Порт
- ПТД
- Новый
- Букинг
- Авто прямое
- Автовывоз

### Запросы без обратной связи
Статус = `Ком предложение отправлено` и `Дата выполнения` старше 3 дней.

### Риск / LOST по клиентам
Берется максимальная дата из:
- Дата последнего заказа
- Дата последнего запроса

Пороги:
- Регулярный: риск с 15 дней, LOST больше 25 дней
- Стабильный: риск с 30 дней, LOST больше 40 дней
- Нерегулярный: риск с 50 дней, LOST больше 60 дней

### Группа A
Если клиент группы A попадает в LOST — подсвечивается критически.
